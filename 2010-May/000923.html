Hi
After wrecking my own system several times, someone suggested using VirtualBox
as a dev platform, which I tried but found problem accessing devices.
So I tried VMplayer, which is free, and found that I can grab the sound card
input. So I can take and it looks like control the M Audio Delta 66 card.
I think I'm not 110% sure that I can also burn CDs from the platform,
this makes VMplayer a very useful emulator, I have FLDIGI running without
a problem, so does WSJT, sdr-core compiles OK Just sdr-shell-4IF to compile and
run.
Pulse audio needs to be turned off. which ever sound card you have on the host
system will appear as /de/dsp, in WSJT this is device 0.
I've found very little difference in the time compiling apps, but playing with
pulseaudio did slow it down.
