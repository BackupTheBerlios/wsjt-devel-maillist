> Bil
>
>> They simply allow '/dev/rig' as the CAT device and the user is expected to make a symlink to the actual device.
> But /dev is reconstituted on reboot and the soft link is removed.  Several Mac users tried this and complained about the need to re-establish the link until I pointed out that the UserGuide explicitly told them how to extend the CAT list thus making meddling with /dev unnecessary.
>
> So the link has  has be redone again, and then again.   Don't you think this is clumsy?   I don't agree with the suggestion.
Ah OK, there are similar issues with Linux since distros started using 
udev for device management. In that case it's pretty straightforward to 
add a custom udev rule to reincarnate the custom device on reboot.

Looking for a similar feature on OS-X doesn't reveal much of any use.

It is possible to have editable items in a combo box, that is probably 
the best way to allow for a custom device without having users delve 
into configuration files. I'll have a go at adding that to my current 
configuration changes with the user entered item(s) being saved in the 
.ini file.

I have looked at enumerating the actual available serial devices but 
this is a whole can of worms in itself (with a nested can of more worms 
when hot-plugged devices like USB get included), I will also revisit 
that when I get the basic rig control working the way I think it should.
>
