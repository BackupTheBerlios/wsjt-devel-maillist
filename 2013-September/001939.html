Bill,

I like the general layout in that it merges the various Configuration/Setup options into a coherent set of panels.   One useful change is removing the confusion (that has been reported to me) over the difference between CAT Port and PTT Port.   Renaming Cat Port -> Serial Port  and PTT port -> CAT port  is more logical.   

However, this means changing naming conventions in  wsjtx.ini   .   What do you propose here?    It means that an older wsjtx.ini cannot be imported to overwrite the new one.  

At present,  CAT ports (later Serial Port) list a lot of COM or USB options, but until I added "User" this list was not user-extensible.   ("User" can be replaced by an addition in wsjtx.ini - see page 5 footnote in User Guide.)   Something like this needs to be added to PTT port (later CAT port). 

I would prefer that instead of having to edit wsjtx.ini to add a user specific driver to the serial list, there should be a panel/box where additions can be made.  Same for the PTT/CAT box.    Ideally, users should not have to edit wsjtx.ini at all.   User definable options/additions should be on the configuration panel
