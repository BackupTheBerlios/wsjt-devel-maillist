Hi All,

I think I've narrowed this down to a missing package. If some of you that are having this trouble could check their install, would be appreciated.

I built a fresh Ubuntu 12.04 + Update, Upgrade and Dist-Upgrade.

Packages added after updates:
sudo apt-get install build-essential, gfortran, subversion. 

Added the Qt5 PPA, then installed Qt5 packages
sudo add-apt-repository ppa:ubuntu-sdk-team/ppa
sudo apt-get update
sudo apt-get install qtbase5-dev qtmultimedia5-dev libpulse-dev libhamlib-dev

Then I ran the Launchpad Test: ./test y
It failed with res: "null".

I started searching qtmultimedia for pulseaudio packages and found ( libqt5multimedia5-plugins ) was not installed. Installed the plugins and ./test y passed.

Then
svn co new r3546
cd ./wsjtx/lib
make -f Makfile.linux <-- ran no errors
cd ../
export QT_SELECT=qt5 <-- This is annoying to do each build
qmake <-- no errors
make -j3 <-- passed no errors

r3546 ran with only (1) error at start-up, which was the palette error, fixed by copying the folder from the source tree.

On the configuration screen, audio was back to normal. Will keep chasing this one through the different UB distros as time allows.
