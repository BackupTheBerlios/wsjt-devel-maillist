There is an auxiliary program that wsjt-x used for decoding called 
kvasd, we don't build it. The CMake scripts should download the correct 
version from the net as part of the 'make install' target. Is the 
message saying that 'kvasd' is missing or is it 'kvasd.dat'?
>
> The second thing, which is more of a problem for me is, Tx power 
> levels seem really low. I have to have the slider all the way up and 
> the Navigator at about 12 O'Clock to get near 25W on JT65 .. normally, 
> or on my Windows box. those setting would generate stupid amounts of 
> TX PWR. Im not sure if this is a Linux or Wsjtx thing. Maybe others 
> have seen it as well, not sure.
I think you are saying that the o/p is lower on a Unix machine. It might 
be that the main audio system levels are reduced. If you run the mixer 
program ('pavucontrol' with PulseAudio on Linux) you can check the 
master levels. Definitely don't worry about setting the o/p level all 
the way up on the wsjt-x interface, it is an attenuator in the digital 
domain so is dividing the sample values which reduces resolution. 0dB 
attenuation is the best place to start and only reduce it if you don't 
have analogue audio settings further down the line.
