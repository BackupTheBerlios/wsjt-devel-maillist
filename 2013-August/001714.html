Hi Bill et al,

I took a quick look at the Qt pulseadudio plugin source code and found that
although pulseadio provides a mechanism to set a stream name, Qt does not
expose it and generates an automatic name.

if (m_streamName.isNull()) // <== This will always be null!
        m_streamName =
QString(QLatin1String("QtmPulseStream-%1-%2")).arg(::getpid()).arg(quintptr(this)).toUtf8();


I have filed a bug with Qt Project regarding this issue.
https://bugreports.qt-project.org/browse/QTBUG-32842
