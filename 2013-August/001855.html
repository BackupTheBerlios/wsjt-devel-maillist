Just committed an addition to on_actionEnable_DXCC_entity_trigger() 
which alters the proportion between the two text display windows 
depending on whether you've enabled the DXCC feature or not.  Took a lot 
of doc searching to find but, as is often the case, the implementation 
is only a few simple lines of code.

The proportions are hard coded but work OK on Linux using the default 
fonts.  Not sure what it'll look like on other OSs.  Its only a 5% bias 
to the left window so shouldn't have too much effect anyway.  Let me 
know if you think the proportions could be improved.
