On 25/08/13 12:05, Murray Curtis wrote:
> Hi all
>
> Revision 3564 allows those with large screens to use bigger fonts.
>
> The decoded text lines should no longer get pushed off the left side 
> of the text browsers mucking up the column alignments if the font is 
> too large - they should now remain left justified with the right end 
> cut off instead.
>
> The main window can now be expanded out wider to allow more room for 
> bigger text too.
>
> A new class DecodedText handles much of the interrogation of the 
> formatted text from lib/decoder.f90.  By having the relevant text 
> columns defined in one place hopefully we'll avoid any obscure bugs 
> caused by the wrong text fields being hard code referenced incorrectly 
> in MainWindow.  I'm slowly moving such code out of MainWindow making 
> it much easier to understand and hopefully easier to maintain.  You 
> may notice in MainWindow::doubleClickCall() there are many commented 
> out bits of code - its still a work in progress so I've left them 
> there for the moment.  I'm a little confused over the cases where CTRL 
> is required for changing the tx frequency but that can wait until Joe 
> is back.
>
> Cheers
