Hi All

John G4KLA found a problem with the blank line between decode periods 
logic.  The conditions to insert the separator include jt9com_.nagain 
which was being set to some odd values as it was passed between the 
fortran decoder and gui threads.  This only occurred on his Mac.  Adding 
write statements to the fortran code would change this behaviour.  It 
would appear that jt9com_.nagain (and maybe others in that block) is 
being overwritten somewhere.

To fix the blank line separator under Mac I've removed jt9com_.nagain 
from the conditions required and instead have set m_blankLine to false 
in on_DecodeButton_clicked() to effect the same logic.

That fixes the immediate problem but we may still have a rogue pointer 
or suchlike somewhere...

Many thanks to John for all his detective work.
