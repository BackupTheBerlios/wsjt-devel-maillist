Both the receive and transmit streams in r3537 have names like
QtmPulseContext, and the transmit audio level needs to be reset at each
transmission.  That keeps me a lot busier than I'd like.......

Back under "portaudio", my linux FC19 was actually using the alsa
pulseaudio plugin.... and I had a reasonably named ("wsjtx") receive audio
stream and no problem maintaining the audio level settings on the transmit
stream.

A while back, I did some integration of pulse audio into QS1R's SDRMAXV
using the multi-platform RtAudio framework (
http://www.music.mcgill.ca/~gary/rtaudio/).  It allowed reasonable names
for the streams.

There should be a way to do this in the Qt Multimedia framework.  If not,
RtAudio seemed to work well in Qt-compiled applications.
