>> I think you are saying that the o/p is lower on a Unix machine. It might be that the main audio system levels are reduced. If you run the mixer program ('pavucontrol' with PulseAudio on Linux) you can check the master levels. Definitely don't worry about setting the o/p level all the way up on the wsjt-x interface, it is an attenuator in the digital domain so is dividing the sample values which reduces  resolution. 0dB attenuation is the best place to start and only reduce it if you don't have analogue audio settings further down the line.
