Hi All,

Qt5 adds some really nice features including a new QObject::connect() 
set of overloads that are far more flexible and efficient. They allow 
type safe connections (i.e.checked by the compiler) and connections to 
arbitrary functions or function objects (allowing use of the powerful 
C++03 binders and function objects).

Since we are still making efforts to support Qt4 and there is a 
suggestion that the C++ Standard library should not be used (I 
understand that where Qt provides matching functionality, but note that 
Qt is clearly moving away from this by extending their containers to 
provide Standard Library style iterators); I propose that we add the 
following small library to the code base:

http://kdemonkey.blogspot.co.uk/2013/06/qt-signal-tools-02.html

This library adds a small number of helper functions and classes that 
emulate most of the new Qt5 connect features in Qt4. The library is BSD 
licensed and could be simply added to our source code or we could make a 
it prerequisite for building, there is one CPP file and some headers. 
Since the makefile doesn't produce a library, I assume the source is 
intended to be added directly to a project that uses it, which the BSD 
license allows.

To test it out do:

cd <somewhere-in-your-sourec-tree>       # I use ~/src
git clone https://github.com/robertknight/qt-signal-tools.git
cd qt-signal-tools
qmake -makefile qt-signal-tools.pro
make
tests/tests

Thoughts?
