I have decided that one of my suggestions is more trouble that it is 
worth. This is only relevant to developers using MinGW on Windows. I 
suggested adding the test install directory to the end of the 
CMAKE_PREFIX_PATH, this is handy if you have intermediate build products 
that are needed later in the install phase. But it tends to make the 
install configure phase unstable because we install some pre-requisites 
directly, this means that the CMake find_... commands can find the 
prerequisites in the install area even if the original source has 
disappeared.

So I now recommend that you do not do this, edit you toolchain file as 
shown below:
>
> C:\Tools\Qt\5.1.0\mingw48_32>type 
> \Users\bill\Documents\objdir\Toolchain-mingw48_32-Qt5.cmake
> # the name of the target operating system
> set (CMAKE_SYSTEM_NAME Windows)
>
> set (QTDIR C:/Tools/Qt/5.1.0)
> set (FFTWDIR C:/Tools/fftw-3.3.3-dll32-2)
> set (HAMLIBDIR  C:/Tools/hamlib-win32-1.2.15.3)
>
> # where to find required packages
> set (CMAKE_PREFIX_PATH ${QTDIR}/mingw48_32 ${FFTWDIR} ${HAMLIBDIR} 
> ${HAMLIBDIR}/bin)
>
> # where the target environment located
> set (CMAKE_FIND_ROOT_PATH  
> ${QTDIR}/mingw48_32*$ENV{HOMEDRIVE}$ENV{HOMEPATH}/Documents/test-install*) 
>
>
> # adjust the default behaviour of the FIND_XXX() commands:
> # search headers and libraries in the target environment, search
> # programs in the host environment
> set (CMAKE_FIND_ROOT_PATH_MODE_PROGRAM NEVER)
> set (CMAKE_FIND_ROOT_PATH_MODE_LIBRARY BOTH)
> set (CMAKE_FIND_ROOT_PATH_MODE_INCLUDE BOTH)
>
> C:\Tools\Qt\5.1.0\mingw48_32>
