Ah, well that might have been deliberate ;) Since the cmake scripts 
don't directly change the product I didn't bother changing 
mainwindow.cpp. But in return the cmake scrips generate a configuration 
header and an SVN version header which is generated at build time. 
Currently I am forcing the compiler to inject the headers into every 
source compile. That is not portable but will do for now until any 
decision is made about CMake. So WSJTX_VERSION_MAJOR WSJTX_VERSION_MINOR 
(defined in the cmake script) and SVNVERSION are defined in all 
compilations.

So using CMake this way there will not be any need to remember to update 
mainwindow.cpp for no reason.
