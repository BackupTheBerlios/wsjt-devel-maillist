That should be it.
I do a:
AC_CHECK_LIB(fftw3f, fftwf_mkplan, \
        [HAS_FFTW3FLIBS=1], [HAS_FFTW3FLIBS=0])
in configure.

% nm -na /usr/local/lib/libfftw3f.a |grep fftwf_mkplan |grep T
00000080 T fftwf_mkplan

so for you
% nm -na /usr/lib/libfftw3f.a |grep fftwf_mkplan |grep T

See if  you get something similar.

The shim library has an extra _ added to the end so fortran can see it,
it then calls the equivalent c function without the extra _
i.e. fortran links with sfftw_plan_dft_1d__ then calls the c function
sfftw_plan_dft_1d_

% nm -na /usr/local/lib/libfftw3f.a |grep sfftw_plan_dft_1d
00002060 T sfftw_plan_dft_1d__
000020b0 T sfftw_plan_dft_1d_

It is possible it is different on other systems, YMMV.
I hope not. *sigh*
