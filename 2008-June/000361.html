Debian update servers in Bulgaria are obviously too slow. So today I
update wsjt by hand. The last build version of deb binary wsjt
(5.9.7.r383-1.3) suprised me as it works stable already several hours in
modes FSK441 and JT65B. Previous package always ended with segmentation
fault. What changed in the package?

Two new bugs though:
1. WSJT can not find KV decoder. Error message:
Error in KV decoder, or no KV decoder present.
Return code:   32512.  Will use BM algorithm.

Is it connected with KVASD compilation and KVASD needs libg2c.so.0 which
is not found any more in Debian. As KVASD_g95 and KVASD_gfortran do not
need ling2c.so.0 shared library perhaps Joe could recompile KVASD too.

2. Common bug in all WSJT releases compiled under Linux. I just did not
mention it earlier as I do not use JT6M. While in mode JT6M and "Auto is
ON" program crashes with this error:

At line 22 of file synct.f
Fortran runtime error: Array reference out of bounds for array 'c',
upper bound of dimension 1 exceeded (330751 > 330750)

Hope this REPORT helps.
