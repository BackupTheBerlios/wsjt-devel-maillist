On Thu, 2008-06-05 at 17:21 -0400, Joe Taylor wrote:

> 1. Remember that file names are case sensitive.  The command invoking 
> kvasd_g95 should be something like
> 
> 	iret=system('KVASD_g95 -q > dev_null')

Yes wsjt_5.9.7r789 works. What I did. Changed the name of kvasd_g95 to
KVASD_g95. Then chmod+x KVASD_g95. The last did not work to execute
KVASD_g95. So I had to change line 13 of the file runqqq.F90 from:
  iret=system( 'KVASD_g95 -q > dev_null')
to
  iret=system( './KVASD_g95 -q > dev_null')

Next I compiled WSJT using g95 compiler:
./configure --enable-g95

Fortunately KVASD_g95 does not need additional libraries like KVASD,
which are not installed here.

Thanks Joe! I have now working copy of WSJT and can wait the code change
while enjoying MS and Tropo JT65.
