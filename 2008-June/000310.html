That is a problem when you link with the statically produced libportaudio.a
on linux only. I have not had time to track this down nor do I think
it is going to be worth it. It is a problem with libportaudio
pulling in some reference to something it shouldn't, i.e. last time
it was jack, I don't know what has changed.

My advice is to put the libportaudio v19 include files into
/usr/.../include/portaudio2 (FreeBSD puts them into
 /usr/local/include/portaudio2 ) and the libs into /usr/.../lib/portaudio2
(FreeBSD puts them into /usr/local/lib/portaudio2)

This will allow you to compile and run programs which are still based
against the v18 portaudio ABI/API.

Then during configure

./configure --with-portaudio-include-dir=/usr/local/include/portaudio2 \
 --with-portaudio-lib-dir=/usr/local/lib/portaudio2 --enable-gfortran

Thus linking in portaudio v19 dynamically instead of statically, this
will bring in all the libs portaudio v19 is expecting.

Note you will have to change the arguments to --with-portaudio ...
to your local site locations.
