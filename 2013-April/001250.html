Hi Joe, All, 

Build OS - Ubuntu 12.04.2 w/Updates, WSJT v0.95 r3240  from a new co from svn.

Build Image: http://imagebin.org/255739

I know you all are about to release on Windows so I thought I'd share a few notes. I'm trying to find things that cross over, so if one gets fixed as will the other.

Hamlib. I opt'ed to just create a folder inside of the source tree, [ WSJTX/wsjtx/hamlib ] and placed the files in it:
 config.h
 rig.h
 rigclass.h
 rig_dll.h
 rig_list.h

Beforehand, I was editing the .cpp and .h files to match the systems install locations, this second method was much easier. I didn't see rig_dll or rig_list specifically being called or included, but placed them in hamlib anyway.

QWT: Packages installed: libqwt6 and libqwt-dev


The location needs updated or linked properly in Makefile.linux, as it's pointing to /usr/incluse/qwt-qt4 the Ubuntu package installs to /usr/include/qwt.

WSJTX.PRO: Needed a few changes due to package locations:

INCLUDEPATH += $$quote(/usr/include/qwt)  <-- updated for correct qwt location
LIBS += -lportaudio -lgfortran -lfftw3f -lqwt <-- updated for correct qwt location.


Creation of JT9 decoder. In the Makefile.linux, if wsjtx_install directory does not exist, the command to copy JT9 over to the install directory, actually creates a "file" call wsjtx_install. Just make sure you have a diretory called wsjtx_install before running Makefile.linux, and all is well.


Geometry and window sizes look nice, but I noticed one thing on the Configuration tab, the COM-PORT select window is too small, as it cuts off the port number.

The .asoundrc patch to set 48000hz in my $USER home directory.

Also, I still have the shared memory issue, so added sysctrl patch and all seems well.

Need to test the new features, but things are looking good, waiting patiently for the Windows release :-)

73's Greg, KI7MT
