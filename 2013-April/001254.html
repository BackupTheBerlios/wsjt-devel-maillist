Hi Edson and all,

Thanks for responding to the comment on wsjtgroup from G8HLH.  These not 
unexpected issues are the main reason for the comment

"CAT control diagnostics are presently rather rudimentary, we're still 
working on this"

that I put into the ChangeLog.  As you say, we need to put some effort 
into dealing with communication timeouts and other exceptions being 
thrown by hamlib.

It seems that having WSJT-X set the radio to USB mode was a bad idea -- 
many people want to use a "data" mode instead.  And the K3, in 
particular, seems to be a bit touchy about timing of CAT commands. 
Possibly polling the radio for frequency every second is too fast for 
some radios.  (It works fine on my TS-2000.)  Too bad the callback 
method does not work.

With some small changes along these lines, and sorting out the exception 
handling, I think most people will be happy.  I'll have some time to 
work on this later today.

	-- Joe, K1JT
