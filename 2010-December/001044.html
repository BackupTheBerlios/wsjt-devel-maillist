> The problem lies with dependencies. At some point, Mac users (and Linux
> too, I suppose) have to get/install:
> python infrastructure including numpy; Imaging; portaudio; fftw3;

Thanks for your comments.  They are just about as I expected.

You are correct about the difference between our present expectations 
for Linux users and Windows users.  The size of the distribution files 
tells the story well: the installation file WSPR2.21_r2286.EXE is 12.0 
MB, while wspr_2.21r2286_i386.deb is only 0.3 MB.

If I knew how to make a "real" .deb file including information on 
package dependencies, it should be possible for the installer to do 
everything automatically.  As it now stands, Linux users must themselves 
see that all prerequisites are installed.

Several years ago Hamish Moffat, VK3SB, made a complete WSJT package. 
At that time, any Ubuntu user could install WSJT by simply doing

$ apt-get install wsjt

or the equivalent in one of the GUI package managers.  For some time 
now, this capability has not been updated.  Is anyone interested in 
doing it again?
