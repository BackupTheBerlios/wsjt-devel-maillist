Hi John, Sivan, and all,

We now seem to have a dozen or more WSPR users using frequency hopping. 
  Probably we'll attract more by the weekend.  Perhaps by next week 
we'll know enough about how it is working to make an informed decision 
about what to keep in the new code, and what to drop.

For example, if coordinated hopping seems best under all (or nearly all) 
situations, we could make it always effective and remove the code that 
selects bands at random.  Your opinions?

Also: should we stop calling this a "beta" and make a new full release? 
  If so, what to call it?  There are now quite a number of enhancements 
over WSPR 2.0.  Should it be called WSPR 2.5?  3.0?

Alas, there should probably be a new manual, too -- with some 
instructions about band hopping, I/Q mode, etc., all in one document.

Other thoughts?
