Hi all on the Developer's list,

I have bumped the WSPR version number to 3.0 and created a new "WSPR 3.0 
User's Guide" to replace the older Guide and its Supplement.  The new 
documentation incorporates instructions for I-Q mode and Frequency 
Hopping -- both of which are significant advances since WSPR 2.0 -- as 
well as a number of relatively minor improvements.

Because several people have asked for it, I've modified the source 
encoding/decoding routines so that power levels less than 1 mW (i.e., 
negative values of dBm) are supported in standard Type 1 messages.
Negative dBm's are *not* supported for Type 2 and Type 3 messages (the 
ones that convey compound callsigns and 6-digit grid locators).

Transmission of negative dBm values should be used sparingly until most 
people have converted to WSPR 3.0.  Messages with negative dBm values 
will not be decoded correctly by older versions of WSPR.

Installation files for WSPR 3.0 have been posted at these URLs:

Windows:
http://www.physics.princeton.edu/pulsar/K1JT/WSPR3.00_r2290.EXE

Linux:
http://www.physics.princeton.edu/pulsar/K1JT/wspr_3.00r2290_i386.deb

The new online User's Guide is available at:

http://physics.princeton.edu/pulsar/K1JT/WSPR_3.0_User.pdf

I will greatly appreciate any quick feedback on the new User's Guide, 
and of course on WSPR 3.0 itself.  I would like to make an open release 
within a few days.

	-- 73, Joe, K1JT
