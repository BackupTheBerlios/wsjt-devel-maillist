Hi Joe,

 > With pctx >= 50 you should always get 3 Tx periods in 2 hours.
 > With 33.3 < pctx < 50 you should get 2 or 3;
 > with 16.7 < pctx < 33.3 you should get 1 or 2;
 > with pctx < 16.7 always 1.

Thanks for the clarification.   I upgraded rxtxcoord to rev2287 and  
changed pctx from 33 to 49 to test the next higher range (33.3 < pctx  
< 5).   Results below for 4 two hour periods with 10 band selection  
but RX only on 160m, 80m and 60m.     (Starting 04:00 UTC to 12:00 UTC)

40:    2     4     2     2   Avg:  3.3
30:    3     2     3     3             3.7
20:    2     3     3     3             3.7
17:    3     2     3     3             3.7
15:    2     3     2     3             3.3
12:    3     3     3     3             4.0
10:    2     4     2     3             3.7

So the fixes to rxtxcoord from rev2286 to 2287 are working nicely.
