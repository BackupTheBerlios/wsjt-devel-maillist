Hi,

I previously reported a problem with rigctl which did not exit with a  
non-zero status after a failed attempt to change bands (with the  
possible result that an illegal transmission could be made, or a  
transmission made to an antenna which is intended only for RX).

Stephane F8CFE at Hamlib was quick to pick up my report (thanks) and  
this has now been fixed.    For those who are concerned about this, a  
new tar file is available from the hamlib project users site:

http://n0nb.users.sourceforge.net/   where you can download   
hamlib-1.2.13svn-rev-3009-20101202.tar.gz

I have tested this with wspr-v2.20 and errors are correctly reported  
and properly handled by wspr code.
