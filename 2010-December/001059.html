Hi Joe,

I have just two comments on the User Guide ..

a)  page 3 para 10.  Another way for Windows users to get accurate  
time is to use regedit to change the "SpecialPollInterval" in the  
NTPclient from 1 week (ie 604800 secs) to a more suitable value - say  
1800 secs.

Add:    " Mac OS X users should ensure that "Set date and time  
automatically" is checked in the Data & Time panel.    If you think  
that the clock is drifting in time, reduce minpoll and maxpoll in / 
etc/ntp.conf and restart ntpd.   The ntp server will be accessed at  
intervals no longer than 2^^maxpoll seconds.   The default setting of  
maxpoll maybe too large."

b)  page 11.  addition to the first paragraph:  It would be help to  
give guidance on the choice of  TX fraction.   In an earlier email to  
mentioned:

With Tx Fraction >= 50 you should always get 3 Tx periods in 2  
hours.  With 33.3 < TX Fraction < 50 you should get 2 or 3; with 16.7  
< TX Fraction< 33.3 you should get 1 or 2; with pctx < 16.7 always 1.

Perhaps this could be included in that paragraph somwhere.
