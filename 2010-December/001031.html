Hi Joe,

I ran tests to find sched_get_priority ranges  if SCHED_OTHER is used

On Mac OS X:      15 to 47  (following the POSIX standard  for  
SCHED_FIFO were at least 32 priority values should be defined).
On Linux:              0 to 0    (SCHED_OTHER - the standard does not  
require that there be defined priorities)
                                1 to 99  (SCHED_FIFO)

If WSPR is to be platform independent, then it might be best to use  
SCHED_FIFO (rather than SCHED_RR) and not SCHED_OTHER.

It might also be best to decide on the middle range first by using   
sched_get_priority_max  and  sched_get_priority_min  and taking the  
mean.   Then raise and lower thread priority around that mean  
value.    This would then work correctly on different platforms with  
different implementations of threading code.

(I use this scheme in multi-threaded code I have written for Linux  
dealing with real-time data acquisition....)

  Comments Sivan?
