On Thu, Dec 9, 2010 at 10:59 PM, Joe Taylor <joe at princeton.edu> wrote:

> Hi John, Sivan, and all,
>
> We now seem to have a dozen or more WSPR users using frequency hopping.
>  Probably we'll attract more by the weekend.  Perhaps by next week we'll
> know enough about how it is working to make an informed decision about what
> to keep in the new code, and what to drop.
>
> For example, if coordinated hopping seems best under all (or nearly all)
> situations, we could make it always effective and remove the code that
> selects bands at random.  Your opinions?
>

I think that the only problem with coordinated hopping is that it will lead
to saturation of the 200Hz bands faster than with random hopping. If many
stations take up hopping, this crowd will move together from band to band,
so we'll see congestion as it gets larger. If stations hop randomly, the
bands will able to sustain a much larger number of transmitting stations.

I don't think it's an important issue for now, and I am not sure if it's
worth keeping the checkbox for it. But I think that this is the fundamental
issue that we need to think about.
