> The first step was to get it linking the .a on i386 first, then work out
> how to modify the configure in portaudio-v19 to get it to produce PIC code
> for amd64 etc. and other architectures.

portaudio's configure has a --with-pic flag which may help to tweak the
way the static library is compiled. I haven't tried it; Debian (and
Ubuntu) has a proper shared portaudio 19 so it's appropriate for me to
use it.

Harry, why not build a static libportaudio.so and include it in your
wsjt package? You can set $LD_LIBRARY_PATH in your wrapper script so
that the linker can find it. Then you can use the shared library but not
depend on one from the distro.
