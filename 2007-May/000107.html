gmtime_r is a POSIX standard library call, and asyou rightly point out
is thread safe by virtue of taking two parameters. On a HP-UX system,
its definition is only available when the -DREENTRANT flag is set for
the compile, which is also done with the -mt compiler flag. It's
possible that similar flags/defines are needed with gcc.

WSJT is a multi-threaded program and unless you're willing to go
through the whole source to check for conflicts in using gmtime() with
its statically defined buffer, then I suggest that the original problem
in getting gmtime_r() linking should be looked at.

Multi-threading is not hugely difficult but old and badly thought out
library calls are good ways of ensuring that it fails in interesting
ways.
