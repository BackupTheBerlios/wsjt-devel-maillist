Hi Richard,

Link from .wsjt to KVASD was used in test rpm which I made for Mandriva
a long time ago. In the newest rpms KVASD is copyed to .wsjt folder.
Here is extractions from wsjt-wraper:

if [ ! -f $RUNDIR/KVASD ]; then
	cp /usr/bin/KVASD $RUNDIR/KVASD

If you ever install Mandriva test rpm it provides the symbolic link to
KVASD hence the problem now. Other posibility is that after installing
wsjt.rpm you first started wsjt as a root being in the user home folder.
Solution is simple in both cases. Simply delete .wsjt folder and start
again WSJT as an user. All should be fine later on.

73, Harry LZ1BB
