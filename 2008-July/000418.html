I am sorry to break you but it will not be so easy to make a deb package
for Ubuntu 8.04. The reason is python-numpy ver 1.1.0 used in Ubuntu. So
one must be developer to change the code of WSJT to fix python-numpy
problem or to wait until the wsjt code is fixed by developers. Becide
python-numpy package there is  another problem with gfortran now. Not a
big deal as a deb maker could use g95. I used g95 in the rpm package
which was proved to work on several major past rpm based distributions
But in any case the posibilities are two. One to be good developer or to
wait untill wsjt code is fixed according the new distribution's
requirements.
