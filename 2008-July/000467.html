Changes to WSPR and WSJT over the past day or two have been 
small, mostly correcting some minor bugs and inconsistencies.

As of r964, I think we are about ready for a public release 
announcement.  If you can compile and test and/or operate 
both WSPR and WSJT from code at the head of the SVN, please 
do so now -- and let me know if you see anything wrong.

My plan is to announce the release of WSPR v1.0 and WSJT 
v7.0.  Since it includes the largely untested WSPR QSO mode, 
I'll call WSJT a beta release; I would like to get some user 
feedback before we say WSJT 7.0 is the full "current 
release".  We should make snapshot tarfiles of the code 
corresponding to these releases.  Installable Packages for 
Windows, various Linux flavors, FreeBSD, and OS/X (two 
flavors?) can then be made from the contents of those 
tarfiles.

MAP65 is also at a point where a new release makes sense 
soon.  Here the number of users (and programming 
contributors) is much smaller ... but if you have any 
opinions or suggestions, I will welcome them.

Does this plan make sense to everyone?  OK to proceed as 
soon as tomorrow?

	-- Joe, K1JT
