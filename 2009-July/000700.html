Harry, John, and all,

Thanks for your reports.

Messages like this one

> "azdist0" requested mutex when "savedata" owned it.

are diagnostics intended to help me understand some of the 
interaction between threads.  You can turn them off by going 
to line #4 of thnix.f90 (or thcvf.f90, if using the CVF 
compiler) and setting ltrace=0.

Similarly, by setting ltrace=3 you can get some more verbose 
diagnostics.  Harry, I suggest that you set ltrace=3, 
re-compile, and let me know if you get any messages before 
the segmentation fault.

I see no segfaults on Linux here, but I've tried it only on 
Ubuntu 9.04.

> There is an anonymous error:   "" requested mutex when "spec" owned it"  .

That was a mistake, I forgot to fill in the routine name in 
one place.  It has been fixed.
