Hullo Joe,

I updated to 1283 but, as in 1277, I occasionally get this error

Error in KV decoder, or no KV decoder present.
Return code:   35328.  Will use BM algorithm.

which I have never had in rev.1105 or earlier.

In addition, there is a problem in extract.f

             if(first) write(*,1000) iret
  1000       format('Error in KV decoder, or no KV decoder present.'/
      +         'Return code:',i8,'.  Will use BM algorithm.')
             ndec=0
             first=.false.
             go to 20
          endif
          read(22,rec=2) nsec2,ncount,dat4
          call cs_unlock

so that when this error occurs there should be    call cs_unlock    
before  the go to 20.   Otherwise jump to 20 skips the unlock and the  
system hangs on a mutex forever.

That is a simple fix.  More importantly, I am puzzled as to why the  
error in kvasd is generated in the first place....

What is error   35328?    Any ideas?
