James writes:

"For re-entrant code, should you really be using the "save" fortran  
statement?"

In particular, when the save statement does not specifically refer to  
named variables, fortran rules indicate that all variables -  
including common blocks - will be saved.    This can lead to  
unintended consequences when other parts of the code might make  
changes to variables in common blocks.

There are situations when "save" is needed but with reference only to  
particular variables.   It might be better to avoid the use of "save"  
where no variables are named.
