> ... This way the user programs
> installed do not interfere with the *base* operating system.

I understand perfectly why things are done this way.  Here's
the problem:

By far the most frequent request I get from *nix users is
for a .deb package that will install WSJT or WSPR on Ubuntu.
  Some time ago Hamish Moffatt did a .deb file for WSJT, but
that package is far out of date and Hamish seems to have
disappeared.

I've looked for someone else interested in making .deb
packages (or, indeed, user-installable packages for other
O/S or distributions), but without success.

So I learned enough to make a very simple .deb package
myself.  Already a number of people are delighted with it.
It provides something we did not have, several weeks ago.

By all means, if anyone can implement a better way of making
installable packages for these programs, that are easy to
keep up to date, please do so!
