On FreeBSD it is /usr/local/bin/wsjt etc. This way the user programs
installed do not interfere with the *base* operating system.

> /usr/share/doc/wsjt/WSJT_User_600.pdf

/usr/local/share/doc/wsjt for FreeBSD

> I do start the program by a wrapper named wsjt. It check and if not
> existing creates ~/.wsjt folder and populate it with some necessory

That wrapper is identical to the one I committed for the project and in
fact is used on the FreeBSD package.

You will note the wrapper already checks for a FreeBSD vs. Linux
directory layout.

In fact, the standard Makefile installs according to the prefix given.

Note that python prefers to do its own compiling and linking for the
.so needed. We obviously have a hybrid system. In the end all that
should be necessary is a "python setup.py install".
