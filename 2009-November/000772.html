Hi John and all,

> I am looking at making a WSJT package that can be downloaded to a  
> Mac.    There isn't a pyinstaller for a Mac so the best would be to  
> encapsulate parts of WsjtMod/* together with some relevant  
> libraries.   The question is the definition of "some".

You're approaching this the same way I did.  As far as I 
know this makes sense, but I am anything but experienced at 
making software packages on *nix.

The .deb packages I made for wsjt and wspr assume that these
packages are already installed:

    python-numpy
    python-tk
    python-imaging-tk
    libsamplerate0

I'm sure it's possible to have this dependence built into 
the package, so that these things will be installed 
automatically, if necessary.  I had not figured out how to 
do this step, so I left it up to the user.

> In the DEB package, what basic infrastructure is assumed to be in  
> place on the host system?   Fortran and POSIX thread libraries?    
> Complete python/numpy/tk infrastructure?   Anything else?    If this  
> is the full list, then I could include appropriate advice to Mac  
> users that at least these should be present and the rest will be  
> provided as part of the package.

Yes, that's exactly what I did.

> I assume that specific libraries (portaudio, samplerate, fftw) will  
> be provided as part of the package and, on a Mac, will be downloaded  
> to the appropriate directory.

There are probably more polished ways of doing this, but it 
should work.

I have now retrieved the .deb package made for WSJT 5.9.7
