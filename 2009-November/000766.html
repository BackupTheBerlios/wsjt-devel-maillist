> Hi Joe,

> It is not a good idea to install all files of any Linux programs in one 
> and the same folder. This is fine in Windows but in Linux the developers 
> have created strict rules which file where to be installed.
> 
> Here is an example where I install each of the files of WSJT in the rpms 
> which I prepared ago. The Linux rules are the same for both deb and rpm 
> development. Actually they are the same rules used in the original 
> Ubuntu WSJT deb file.

> Yes the program  will work fine if you install all files in one folder 
> but this is out of the Linux rules. I kindly please you to remove "make 
> deb" command until things are regulated. Other way we will distribute at 
> least bad manners between Linux users. Breaking the Linux rules is 
> actually much worst than bad manners.

> I do start the program by a wrapper named wsjt. It check and if not 
> existing creates ~/.wsjt folder and populate it with some necessory 
> files. Actually this happens only once at the first start. Here is the 
> wrapper which actually I shamesly plagiarized from Hamish, VK3SB:
> 
> #!/bin/sh
> if [ -e /usr/local/bin/python ] ; then
>     LOCALBASE=/usr/local/
> else
>     LOCALBASE=/usr/
> fi
> if [ ! -e ~/.wsjt ] ; then
>     mkdir ~/.wsjt
>     mkdir ~/.wsjt/RxWav
> fi
> if [ ! -e ~/.wsjt/CALL3.TXT ] ; then
>     cp -f ${LOCALBASE}/share/wsjt/CALL3.TXT ~/.wsjt
> fi
> cd ~/.wsjt
> python ${LOCALBASE}/bin/wsjt.py
> 
> I hope this helps.
