Hello Pierre,

I can confirm that rev. 1617 works OK. Do not worry about these messages.
In the directory where you compiled wspr, just start it with the 
following command;

python wspr.py

It should come up, then just make sure your options are set correctly.

If it complains at startup about  audio_caps file, just create one with 
something like  this;

 0   16   16       0       0  /dev/dsp

then restart it and set your options.

I have been running for a few days now on an Ubuntu 9.04 64. Works great!

Bon murmure,

73
de Mike, VE2ExB
