Hi Diane and Joe,

Thanks for confirmation about the fftw3 library and how to fix it.

It would be a help if the Win/Ubuntu version of the library could be  
constructed with the correct Fortran entry points.   This has two  
advantages:

a)  four2a.f would then conform to the usual Fortran convention (of  
not using trailing underscores)
b)  source code would be compiled and linked correctly by a wider  
community - eg including BSD Unix used on Macs, and perhaps over  
versions of Linux.

The alternative would be to have additional #ifdef's in four2a.f to  
distinguish between versions of the library which have conventional  
and unconventional Fortran entry points.
