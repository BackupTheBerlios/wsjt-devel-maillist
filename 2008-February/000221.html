Dear Colleagues,

I have not yet tried to run down the problems, but changes submitted by 
Chris in r627 and r628 have seriously broken WSJT, at least on my system.

I just did an update.  A trivial syntax error had been introduced into 
chkmsg.f, so I fixed that.  The program then compiled, but on startup it 
produces the output appended below.

After doing "svn -r626 update" and then make, all is well again.

This test is on Windows with the MinGW/g95 compile tools.

It is difficult to follow the rationale for the revisions made in r627 
and r628, because the diff files are so long and many (most?) of the 
changes seem to be trivial formatting changes that do not actually 
change the code.  In general, such changes should be avoided.  When you 
change a line of code, the reason for the change should be clear.  If 
there is no reason, don't change it.

Insofar as possible, when you think large changes should be made, it's 
important to do what you can to be sure that they don't break the code 
on systems different from yours.  WSJT is now compiled and run on 
Windows, FreeBSD, many flavors of Linux, and OS/X.

Chris, perhaps you can explain better what you trying to do in r627, and 
why you considered it necessary.

	-- 73, Joe, K1JT
