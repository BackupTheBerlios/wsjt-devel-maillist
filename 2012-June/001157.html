Now this is curious!

I am running Linrad into MAP65 r2471 on the same machine,
with integer TIMF2 signals.
Input into Linrad is from my SDR-IQ.

Suppose I start up MAP65 r2471 first.
Then I start Linrad up.
As soon as I select the mode in Linrad
to bring up the main Linrad screen,
the 2 Doppler values (MyDop and DXDop) in MAP65 change from
sensible and correct (same as WSJT) values
to obviously implausible values!

Stop Linrad and the Doppler values remain bad.
Restart MAP65 and all is well - until I start Linrad again.

I have tried Linrad versions 3.37, 3.38 and 3.39, same problem.
But MAP65 r2465 is ok.
It looks as if something in MAP65 r2471 is being overwritten
by the data stream from Linrad.

Does anyone else have this problem?
