Need some help here Please
I WAS using Debian Lenny Default Kernel 2.6.26-2
I did sudo apt-get build-dep wsjt and also made sure I had the extra 
Dependencies
included as suggested at wsjt-development
In order to make wsjt work from an svn build I had to UNINSTALL the 
Default numpy 1.1.0
and INSTALL/BUILD numpy 1.0.4

Then I used these Commands:
svn co -r 1779 svn://svn.berlios.de/wsjt/trunk -- you can change r xxxx 
to what ever version you want/need
./configure --enable-gfortran --with-portaudio-include-dir=/usr/include 
--with-portaudio-lib-dir=/usr/lib
sudo gedit
go to trunk in my home folder scroll down to makefile and change
WsjtMod/Audio.so: $(OBJS2C) $(OBJS3C) $(OBJS2F77) $(SRCS2F90) $(AUDIOSRCS)
         ${F2PY} -c --quiet --noopt --debug \ <--- put your mouse here 
and backspace
         --f77flags="${FFLAGS}" --f90flags="${FFLAGS}" \
to read
WsjtMod/Audio.so: $(OBJS2C) $(OBJS3C) $(OBJS2F77) $(SRCS2F90) $(AUDIOSRCS)
         ${F2PY} -c --quiet \
         --f77flags="${FFLAGS}" --f90flags="${FFLAGS}" \
click on save
then
make -j2
sudo checkinstall -D
I change
1) to read Summary: wsjt weak amateur radio digital signals
2) to read: wsjt
3) to read Version: r1779
Type enter and the build is make into a .deb package

There will be a nag file appear when you run wsjt from the terminal so 
in order to fix it
bring up a terminal BEFORE you run wsjt and type
cd trunk
cp KVASD_g95 ../.wsjt
sudo su
cp KVASD_g95 /usr/local/bin

Bring up a terminal type in wsjt you should now have a running copy of wsjt7
