I came up with the following idea for addressing the randomness vs. control
issue.

While hopping, we could treat the band menu as a "hop there next" command.
If you choose 20m on that menu, the current cycle continues normally, but at
the next hop opportunitie (the end of the current cycle) it will hop to 20m.
If you also click on Tx next, it will transmit there.

In fact, I think that this is a good behavior even when not hopping; there
is no reason to switch frequencies in the middle of a cycle.

If we do this, I think that the UI will need to change a bit, to indicate
that the user told the program where to go next, like the Tx Next button
shows that.

Sivan
