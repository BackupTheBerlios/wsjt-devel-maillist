I have tried frequency hopping on Mac OS X and it works fine.   Nice  
idea, Silvan.    Joe's addition in wspr.py to call "user_hardware"  
provides additional flexibility to send rigctl commands to make other  
changes as needed.

I'm using a SignaLink USB interface for audio/PTT to a Kenwood TS-870  
and a USB-serial adapter (PL2303) to control the rig frequency via  
its serial port.     The 'user_hardware' file just contains a simple  
(UNIX) shell script to call rigctl to dump the VFO frequency as a  
test of principle.
