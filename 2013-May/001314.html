Hi David,

1. There's no plan to make Qt5 a requirement for compiling WSJT-X, just 
an option.  In fact it's possible already -- Edson (PY2SDR) and I have 
both made it work, with very few changes to existing code.  The results 
are still backward compatible with Qt4.

2. There's no plan to relax portability requirements.  Rig control 
through hamlib calls was first and foremost in WSJT-X, among possible 
CAT control options.  It will remain so.  Rig control through Ham Radio 
Deluxe was added because many people asked for it; likewise for 
Commander, for which coding is still to be done.  HRD and Commander are 
seen as options, not requirements.

3. I haven't messed with Pulse for audio I/O, but surely it is NOT 
platform-portable, right?  (Windows, OS X ??? ) Qt Audio runs on all 
Qt-capable platforms.

	-- Joe, K1JT
