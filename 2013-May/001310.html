No offense taken.

Our top priority here is developing reliable, easy-to-use software that 
hams can use for weak-signal communication.  To that end we make sure 
that we have a click-to-install package for Windows, the most common 
platform in hamshacks.

A second priority is to provide an educational resource for hams who 
know some computer programming and want to learn more about the DSP 
techniques required to implement weak-signal modes like JT65 and JT9. 
That's why the programs are open source -- so other hams can contribute 
to their development now and in the future -- and why we make sure they 
can be compiled and run under Windows, Linux, and OS X.

In due course, we'll likely find time to put together a "configure" 
script for unix-like operating systems so that a user can download the 
source, type something like

$ ./configure
$ make
$ sudo make install

and be ready to go.  In the meantime, compiling necessarily requires 
some knowledge of what is happening, and why... and how to fix it if 
something is not quite right.
