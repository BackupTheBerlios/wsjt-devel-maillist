Hi all,

WSJT-X r3323 has been released as Version 1.0.  I updated the WSJT web 
site, making a new page for WSJT-X, and also the WSJT-X User's Guide.

I have made a new branch called "wsjtx_k1jt_p1" in the SVN repository at 
Berlios.  Its initial contents are in sync with Version 1.0.  I will 
soon start work on adding JT65 as a new mode, working in this branch.

When we make bug fixes or relatively minor enhancements to the 
production code, they should be done in the original "wsjtx" branch.

As a reminder: the tasks we've talked about for future development include:

     - Moving from Qt4 to Qt5, and eliminating dependence on Qwt
     - Adding rig control through DxLab Commander
     - Migrating from PortAudio to Qt's built-in QAudio handlers
     - Adding other WSJT modes: JT65, FSK441, ISCAT, JT4 ...
     - Support for I/Q transceivers such as SoftRock
     - A generalized inter-program interface
     - Expanded frequency range, perhaps 5 kHz or so.

Several additional tasks that I consider desirable:
     - Writing "HowTo" guides for compiling WSJT-X in Windows/MinGW,
	Linux, OS X, ...
     - Making binary installation packages for Linux and OS X

If you want to tackle one of these tasks, I suggest (1) letting us know 
what you're up to, and (2) if programming is involved, making yourself a 
private SVN branch.

The command I used to make branch wsjtx_k1jt_p1 (executed from my 
existing wsjtx directory) is

svn copy https://svn.berlios.de/svnroot/repos/wsjt/branches/wsjtx 
https://svn.berlios.de/svnroot/repos/wsjt/branches/wsjtx_k1jt_p1

	-- 73, Joe, K1JT
