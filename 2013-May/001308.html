If you're not using QtCreator, I don't know exactly what to expect.

Those error messages suggest that you're not configured in such a way 
that the correct header files are being found and included.
Maybe Edson can suggest why this might be so.

So far, I've done the Qt5 compile only in Windows, under MinGW.

There's really no reason for you to be trying to compile under Qt5, yet: 
the program can't run without a Qwt "thermo" object also compiled under 
Qt5, which we don't presently have.  Edson is thinking about making an 
alternative.

My advice is to stick with Qt4 until we get some things straightened out.
