Hello Joe,

After some investigation I have discovered that it is not possible to have
asynchronous calls to the hamlib functions. All functions that communicate
with the transceiver call read() in a blocking mode. This post on the
hamlib-dev list have some information.

http://osdir.com/ml/linux.hams.hamlib.devel/2008-08/msg00001.html

A possible work around is to turn (our) rigclass into a Qt object, run it
on a separate thread, and use Qt's signal/slot mechanism to handle the
asynchronous communications. I need to think a little more about the issue
to see if I can come out with a simple way to do this.

73,
