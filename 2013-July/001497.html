Hi Edson,

PY2SDR wrote:
> r3478 compiling without issues and running fine so far on Ubuntu 12.04.2.
>
> I have a GUI suggestion for consideration: We could split the two decode
> areas into a separate window. This would allow a larger decode area and
> more space for future addictional GUI elements in the control window. The
> decode window and control window could be placed side-by-side and the
> spectrum window either above or below the decode and control windows.

Thanks for sharing your idea.  My view is that it's best to keep the "Rx 
Frequency" window close to the Tx message controls, and therefore on the 
same panel.  Importance of the "Band Activity" window is less immediate, 
and it could be moved to a separate panel.  That would make WSJT-X 
somewhat more like MAP65, which has a separate "Messages" window serving 
the same function as our "Band Activity" box.

I don't want to start major changes just now, though... I'd like to get 
things ready for a stable release of Version 1.1.
