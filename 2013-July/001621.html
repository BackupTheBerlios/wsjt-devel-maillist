Hi Bill,

> 1) The audio output routine appears to allow T/R period and #
> samples/symbol to be changed on the fly mid transmission. Is this really
> a valid thing to do?

No.

> 2) I am struggling to understand the setting of SNR at the start of a
> transmission period and also live changes to it during transmission. It
> appears to be possible to start with it set to 99.0, can you explain how
> SNR changes the current transmission please?

This is an undocumented feature that I've sometimes used for testing the 
decoders.  Random gaussian noise is added to the generated audio signal 
so that it will have a specified S/N.

Specifying a positive SNR (for example, snr=99) is taken to mean "this 
is a normal transmission, don't add any noise."
