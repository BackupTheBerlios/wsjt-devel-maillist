G4WJS wrote:
> I was looking at the Qt 5.1 docs and the QAudioInput class is not
> described in the Audio overview, nor is it listed in the module class
> list. I suppose the docs are still a work in progress. Having found it
> by looking up in the full list of classes; yes it does seem to fit the
> requirement well.

Actually, I've found the documentation to be one of Qt's strongest 
features.

For QAudioInput and QAudioOutput go to

http://qt-project.org/doc/qt-5.1/qtmultimedia/qaudioinput.html
http://qt-project.org/doc/qt-5.1/qtmultimedia/qaudiooutput.html

...

A few hours ago I quickly cobbled together drop-in replacements for 
soundin.h and soundin.cpp that use QAudioInput instead of PortAudio. 
The code compiled and ran without problems, and within an hour WSJT-X 
was decoding both JT65 and JT9 signals with the nes input routines.
The same approach for audio output should not be any more difficult.

I'm not suggesting that others should stop thinking about the best way 
to do these tasks... just reporting that a baseline system (the input 
device selection was temporarily hard-wired, for example) was easy to 
get going.
