I'm not usually envious of Windows users but JTAlert is very very useful 
while working JT65/JT9.  Several times I've answered a CQ that I'd 
already worked, on one occasion only about an hour earlier!  Something 
had to be done..

I have some code that reads an ADIF file and cty.dat.  At the end of 
each decoded CQ call it adds the country name from cty.dat and a single 
character status, '!' for new country and  '~' for new call. If you've 
worked that call before its just the country name.

Heres a picture: screenshot 
<https://dl.dropboxusercontent.com/u/239925/wsjtxAlert.png> 
(https://dl.dropboxusercontent.com/u/239925/wsjtxAlert.png)

Its pretty simple and pretty dumb. Band and mode are not considered (I'm 
only active on 20M).  I don't want to compete with vk3ama; Laurie is my 
(relatively speaking) neighbour after all.  And JTAlert does so much 
more than this.

Is this a feature we really want in WSJTX?   I currently have it 
enclosed by #ifdef UNIX as I figure Windows users won't want or need it 
- as I type this I realize that our Mac friends may though... I'm 
hesitant because I've been caught by feature creep before in other 
projects and WSJTX is currently nicely focused on what it does well.

Thoughts?
