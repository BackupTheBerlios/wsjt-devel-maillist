Dear Colleagues,

An open release of WSJT-X Version 1.1 has been posted on the WSJT web 
site: http://physics.princeton.edu/pulsar/K1JT/wsjtx.html

I've decided to still call this a "beta" release, mostly because I 
haven't figured out how to make Ham Radio Deluxe set the frequency of a 
radio's "VFO B" and use that when transmitting.  (Surely this must be 
possible, but the available documentation is miserable.)

I think a few users are still finding CAT control to be less stable than 
it should be: possibly because a call to "rig->getFreq()" times out when 
the decoder is getting nearly 100% CPU time.

Otherwise, I think this is a stable, solid release.

Tasks now known to be "in the works" (or at least being thought about) 
include the following:

1. Implement a way for CAT control to work in Split mode through HRD.
2. Fix the bug causing occasional loss of CAT control.
3. Substitute a more attractive Thermo control.
4. Fix the (shared-memory overwrite?) problem with running multiple 
instances.
5. Further tests and optimizations of JT65 and JT9 decoders for best 
possible speed and sensitivity.

Additional suggestions for program improvements will be very welcome -- 
as will volunteers willing to work on them, in any capacity.  Program 
code, improvements to our documentation, and detailed testing reports 
are all needed and welcome.

	-- 73, Joe, K1JT
