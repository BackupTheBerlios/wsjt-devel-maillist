On Tue, Jul 9, 2013 at 4:17 PM, Joe Taylor <joe at princeton.edu> wrote:

> Dear Colleagues,
>
> I think WSJT-X r3471, now integrated back into branches/wsjtx, is
> approaching a good state for release as Version 1.1.
>
> I posted a Windows package of r3471 earliet today so that Laurie, VK3AMA,
> could use it to test his new version of JT-Alert.
>
> It has a number of minor improvements and bug fixes relative to r3459. The
> Windows package and can be downloaded from
> http://www.physics.princeton.**edu/pulsar/K1JT/WSJTX_**110r3471.exe<http://www.physics.princeton.edu/pulsar/K1JT/WSJTX_110r3471.exe>,
> and of course you are more than welcome to try it on OS X and Linux, as
> well.
>
> Here's a quick list of differences from r3459:
>
> 1. Operating in "Split Tx" mode is now optional (but of course is required
> if you will transmit above 2.7 kHz.)
> 2. Correct TxMode label now appears on program startup.
> 3. "Transmitting..." message in ALL.TXT now has the correct TxMode.
> 4. File wsjt_status.txt includes the TxMode.
> 5. F4 clears Az, Dist, and the Tx message boxes.
> 6. You can decode JT9 signals in the JT65 frequency range, and ice-versa,
> by double-clicking the signal in the waterfall.
> 7. Better logic for enabling/disabling the TestPTT button.
> 8. Do not call getFreq() when the TestPTT button is clicked and PTTmethod
> is via CAT control.
> 9. New cold-start default frequencies, set at reasonable values for JT65.
> 10. Default macros as simple examples.
>
> This revision (or something close to it) will likely be released publicly
> soon.
>
> Please report any compilation or operational problems!
>
>     -- 73, Joe, K1JT
