On 07/02/2013 05:08 PM, Joe Taylor wrote:

> If you're using WSJT-X v1.1 in OSX or Linux, what are your settings for
> PTT Method and PTT Port?  I'm trying to rationalize the related code.

I'm running Fedora Linux 17. I'm using hamlib rigctld because I'm able 
to use it also when running fldigi. Further, I have to use additional 
arguments when calling hamlib. When using rigcat with fldigi, these 
additional arguments are possible too. When using rigctld instead, it's 
the job of rigctld to manage it. The CAT communication is then only via 
a TCP port on localhost, not more directly on the serial port. When 
using wsjtx, I have to change the config by hand in order to tell wsjtx 
to use a TCP, not a serial port. In this manner, the problem of passing 
additional arguments does not occurs because rigctld is controlling the 
serial port. Of course, the PTT control is via CAT too.

Here is my starting sequence used to start rigctld:

/usr/local/bin/rigctld -m129
--set-conf=rig_pathname=/dev/ttyUSB0,
serial_speed=4800,dcd_type=RIG,timeout=500,
retry=2,write_delay=5,post_write_delay=50,itu_region=1 &

It would be fine if wsjtx would allow to select a TCP in the config, 
instead of a serial port, when rigctld is used.

Many thanks for your explanation about the frequency shift, in my other 
message.
