Nothing about multiple instances is in the v1.1 user manual yet, so I was
winging it.....

I have installed the latest v1.1 r3482 package on
Windows 7, in separate directories:

  C:\wsjtx
  C:\wsjtx2
  C:\wsjtx3
  C:\wsjtx4

Each is configured to look in their own directory, and all have the
Setup->Advanced->Allow multiple instances item checked.
All are shut down after configuration, and all are configured one at a time.

They get restarted in numeric order, and the configuration checks out.
 However, when the second instance starts, the "Decode" button on the first
instance turns on and stays on.  The spots from the first instance then get
reported in the second instance.  They are not shown as decodes in the
first instance.  And the spots are logged in the second instance's ALL.TXT
file, which is in a different directory than the first instance's ALL.TXT
file.

If the spots included the manually entered window frequency offset, they'd
be sorted in the destination ALL.TXT file.  But they only have the audio
frequencies, so are all jumbled together.

Is wsjt-x designed to merge the logging/spotting data, or keep it separate
by instance?

Thanks,

A separate instance of r3482, built from source on Linux, is chugging away.
Don't have more radios to throw at that machine, so haven't been able to
try multiple instances on Linux.
