On 07/28/2013 04:10 PM, Edson W. R. Pereira wrote:

> Yesterday I've done a few tests with Qt5 Audio and have some
> observations. On Linux, Qt5 either uses pulseaudio or ALSA as sound
> input/output. Either one or the ohter (not both).

The idea behind pulseaudio is to liberate the application from the need 
to deal with the hardware. The application, as client, requests streams 
from the pulseaudio server. The selection of devices, the mixer, the 
volume control are now a matter which pulseaudio has to manage. There 
are tools available to the user. pulseaudio is now the standard sound 
system of most modern Linux distributions.

Have a look on fldigi using pulseaudio.
