On 31/07/13 06:48, Joe Taylor wrote:
> Hi all,
>
> Revision 3510 includes some very basic proof-of-concept code using 
> QAudioInput instead of PortAudio for sound input.  It builds on Bill's 
> refactored code.  By default, this test code is not active, and the 
> program will compile using PortAudio.
>
> To compile using QAudio, uncomment the line
>
> #DEFINES += QAUDIO_INPUT
>
> in wsjtx.pro.  The files soundin.h and soundin.cpp each have two 
> disjoint sections: the first using portaudio, the second using QAudio.
>
> Note that device selection in the QAudio code is presently
> hard-wired to use the default input device.  You can change this
> choice on line 267 in soundin.cpp, to test on your system.
>
> I have no special pride in this code -- I just made something that 
> works, and dropped it into Bill's code.  Please feel free to make it 
> better.
>
> I suggest that we do something similar for audio output.  When/if we 
> make the changes permanent, we'll need also to modify the device 
> selection code in devsetup.cpp.
>

> work in your Linux system?
