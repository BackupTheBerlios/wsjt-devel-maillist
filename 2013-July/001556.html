When I was looking into the comms issues recently, I built with tools in 
the Cygwin environment. They have a Mingw package with g++ & gfortran at 
4.7 and that seemed ok with Qt 5.1.0.

I did have issues when running under gdb on Windows, the wsjtx exe 
failed to exit cleanly. It SEGV'ed before the settings could be saved 
which was annoying. I didn't manage to find the cause of this. The same 
exe seemed Ok outside of gdb. This might be a toolchain version issue or 
simply an issue with the source itself.

>
> Advice?  Recommendations?

I guess the only thing that matters is if someone who has to build (for 
some obscure platform for example) is restricted to an older version of 
something in the toolchain.

Perhaps those who make binary packages for other than Windows can 
comment on what they use?

Personally, with my C++ programmer hat on, I would prefer g++ >= 4.7 as 
the current ISO Standard support is weaker before that.

My FORTRAN programmers hat was lost and forgotten many years ago (think 
punched card decks passed to operators through a hatch in return for 
dead trees some time later if there weren't any compiler errors).
