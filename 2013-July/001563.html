Hello Bill,

I assume that WSJT-X works with the same toolset as for WSPR-X (Guys, 
please correct me if I am wrong)
I recently did the following for WSPR-X which worked for me:

Install    7zip, to unpack .tar and .gz:
Get        7z920.exe
 From 
http://sourceforge.net/projects/sevenzip/files/7-Zip/9.20/7z920.exe/download?use_mirror=freefr
Install

Get        gcc-full-4.4.0-mingw32-bin-2.tar.lzma  2009-06-24    62.2 MB
from 
http://sourceforge.net/projects/mingw/files/MinGW/Base/gcc/Version4/Previous%20Release%20gcc-4.4.0/
Copy to    c:\MinGw
Unpack in  c:\MinGw using 7zip "Extract here"

Get        w32api-3.13-mingw32-dev.tar.gz
 From http://downloads.sourceforge.net/mingw/w32api-3.13-mingw32-dev.tar.gz
Copy to    c:\MinGw
Unpack in  c:\MinGw using 7zip "Extract here"

Get        qt-win-opensource-4.8.5-mingw.exe
            = Qt libraries 4.8.5 for Windows (minGW 4.4, 317 MB)
>From       http://qt-project.org/downloads
Install    Run qt-win-opensource-4.8.5-mingw.exe
Into       C:\Qt\4.8.5

Get        mingw-get-inst-20120426.exe
>From       http://sourceforge.net/projects/mingw/files/
Install    Run mingw-get-inst-20120426.exe
Into       c:\Gw
            Select all components

g95-MinGW.exe:
I tried to download without success as follows:
Get        Stable Version 0.93, October 2012  Self-extracting Windows x86
>From       http://www.g95.org/downloads.shtml
Fortunately I found that I had a copy from a previous attempt, last year.
I can send that file by email.
Install    Accept suggestion to install in c:\mingw

Settings   Go to Control Panel - System - Advanced - Environment Variables
            Add  New system variable: LIBRARY_PATH, value: C:\MinGW\lib
            Edit system variable PATH, add: 
c:\MinGw;c:\g95;c:\MinGw\bin;c:\g95\bin;c:\svn;c:\qt\4.8.5\bin

Get        qwt-6.0.1.zip    2011-08-01    3.5 MB
            qwt-6.1.0.pdf    <==== Not available?
 From http://sourceforge.net/projects/qwt/files/qwt/6.0.1/
Install    See http://qwt.sourceforge.net/qwtinstall.html
            1) Unzip and copy to c:\
               Result in c:\qwt-6.0.1
            2) See Readme and Install
               qmake qwt.pro
            make  (several warnings)
            make install (several warnings)

Get        pa_stable_v19_20111121.tgz
>From       http://www.portaudio.com/download.html
Extract    to c:\PortAudio


Copy       palir-02.dll, qwt.dll
from       wsprx (downloaded program)
to         c:\QtSupport\

Copy C:\MinGw\lib\gcc-lib\i686-pc-mingw32\4.1.2\libf95.a
To         C:\MinGw\lib\libf95.a

Turtoise SVN
I had to add the following for my Windows XP machine:
Get        WindowsXP-KB942288-v3-x86.exe
 From http://www.microsoft.com/en-us/download/details.aspx?id=8483

Get        tortoisesvn
>From       http://tortoisesvn.net/downloads.html
Install    Include command line


Procedure:
========
Do         svn co -r 3058 svn://svn.berlios.de/wsjt/branches/wsprx
Result     Source code is put in c:\wsprx
Note       Omitting r 3058 will get the latest version

Add        INCLUDEPATH += c:/portaudio/include
Edit       Remove or comment out the line LIBS += -lusb   ( #LIBS += -lusb )
To         c:\wsprx\wsprx.pro
Note       I had problems finding usb.dll and was adviced by K1JT that:
            The usb code is in file libusb0.dll, which is easy to find 
on the web. But in fact I don't think you need        it (except 
possibly for rig-control of a Softrock?).  Just remove (or comment out) 
the line
            LIBS += -lusb

Edit       main.cpp
            #include <QtGui/QApplication> => #include 
<QtWidgets/QApplication>

Do         cd \wsprx\lib
            mingw32-make
Result     Aborts with Error
Do         mingw32-make
Result     Aborts with Error
Do         mingw32-make
Result     Completes with Nothing to be done
            Created wsprd.exe and wsprcode.exe

Do         cd\wsprx
            qmake.exe -o Makefile wsprx.pro
Result     Updated make files

Do         mingw32-make -f makefile.release
Result     wsprx.exe
In         wsprx_install

Replace the wsprx.exe that was downloaded with the K1JT WSPR-X kit with 
the newly built wsprx.exe

Of course I like to hear from others how this can be simplified or improved.
