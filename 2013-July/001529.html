I am stress-testing the changes to support multiple instances
of wsjtx.  Should be done by tomorow evening.

The change involves relatively minor changes to the
following files:
     main.cpp
     mainwindow.cpp
     mainwindow.h
     lib/jt9.f90
     lib/jt9a.f90
     lib/ipcomm.cpp

The calling convention for jt9 changes from
"jt9 -s" to "jt9 -s <shared_memory_key>.
The keys are programatically generated with QUuid and should
be unique for each instantiation of wsjtx and its
corresponding jt9.  Example:

jt9 -s {ec40dd52-d366-4dfe-a885-b4695d43d09d}

How should I submit the changes for review?
