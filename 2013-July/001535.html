Hi David,

G1OXB wrote:
> The "Split" TX mode scheme works with both radios as long as you leave time, about 5 seconds, for a lot of reading
> and writing (seems like more than really necessary) of the A and B frequencies  by the CIV controller
> before manualy setting the SPLIT mode or activating the antenna tuner on the radio.
> Both  of these actions can cause current CAT commands to be aborted and sometimes leave the A / B VFOs
> incorrectly set.

I've no idea why your setup should need 5 seconds or a lot of reading 
and writing of frequencies, but I don't have your hardware.  These 
things take a fraction of a second here.

Why are you setting Split mode or activating an antenna tuner after you 
double-click on a decode to call a station??  These things can be done 
once and for all, when you change bands.

> I have been looking at the source code to see how difficult it would be to compile it myself.
> It looks like a lot of effort.

1. Install necessary software tools
2. $ mkdir ../wsjt_install
3. $ cd lib
4. $ make -f Makefile.Mingw    (or make -f Makefile.linux)
5. $ cd ..
6. $ qmake wsjtx.pro
7. $ make

> Looking at the sources for the first time, they are an odd mixture of
>old-style Fortran and the latest GUI language.

Perhaps you consider the mixture odd.  As we say, there are many ways to 
skin a cat.

I find the code mixture a well-optimized choice that produces a snappy, 
convenient, appealing, and platform-independent GUI together with 
extremely efficient number crunching.
