Hi Claude,

On 7/28/2013 7:05 AM, Claude Frantz wrote:

> I'm using Fedora 19 here now. After the last updates of wsjtx via svn, I
> was not more able to compile wsjtx because Qt 5 is required now. I
> ignore if there is a way to enforce the use of Qt 4.

Probably you have already figured this out, but just in case you have 
not: it's possible to compile WSJT-X using either Qt 4.x or Qt 5.x, with 
nothing more than activating one of the following statements in wsjtx.pro:

DEFINES = QT4
#DEFINES = QT5
