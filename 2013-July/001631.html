On 31/07/2013 19:00, Joe Taylor wrote:

Hi Joe,


> As mentioned in a previous message, I'm recommending that we should 
> soon release WSJT-X version 1.1.1 as a beta.  The list of improvements 
> since v1.1 is already fairly long, and users are clamoring for some of 
> them. At this point, such a release would use the Portaudio code.
>
> Here's a more-or-less complete list of improvements since v1.1 r3496:
>
> 1. New organization of the "Band Settings" tab on the configuration 
> screen allows settings for different antennas on different bands, and 
> these descriptions are sent to PSK Reporter.
>
> 2. Full control is nor provided for DTR and RTS lines on a serial port 
> used for CAT control.
>
> 3. There is now a rudimentary user control of font sizes.  If a file 
> named "fonts.txt" exists in the WSJT-X installation directory, it's 
> assumed to have four integer numbers on a single line.  The numbers 
> are point size and weight for labels, then point size and weight for 
> decoded text.  For example, try the line "9 50 12 100".
>
> 4. A user control at the bottom of Wide Graph window sets the start 
> frequency of the waterfall scale.
>
> 5. There's a new optional palette "gray1" for the waterfall.
>
> 6. The small colored button just right of the "+2 kHz" checkbox now 
> shows the letter "S" if "Split Tx" has been checked.
>
> 7. New user option appends DXCC country name and "QSO B4" status to 
> all CQ messages.
>
> 8. Several minor bug fixes.
>
> 9. Refactoring of code to move audio input into the GUI thread. 
> (Should be invisible to users).
>
> 10. Updated credits displayed in the "About" window.
>
> 11. Updated User's Guide.
>
> Is everyone OK with this proposal?  I suppose it will mean releasing 
> something close to r3514 (Compiled with Portaudio, not QAudio) as 
> Version 1.1.1.
>
> Comments ?
