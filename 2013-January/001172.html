Hi Joe et al

Thanks for the help Joe. Finally got the .exe built on XP and it works just 
fine! I'll detail what I've done while still fresh in my mind... may help 
others:

Platform: Windows XP SP3
Downloaded installation packages:

Setup-Subversion-1.7.8.msi
mingw-get-inst-20120426.exe
g95-MinGW.exe
python-2.7.3.msi
numpy-1.6.2-win32-superpack-python2.7.exe
PIL-1.1.7.win32-py2.7.exe
Pmw.2.0.0.tar.gz
pyinstaller-1.5.1.zip
pywin32-218.win32-py2.7.exe

I used mingw32-make with Makefile.MinGW to build, and also needed to install 
mv.exe using:
coreutils-5.3.0-bin.zip (mv.exe)
coreutils-5.3.0-dep.zip (dll's)

Of course make sure that above installations are pathed to wspr dir...
...and in addition Joe's info:

The following commands check out and build WSPR 3.0.1 on my development
machine:

C:\tmp> svn co -r2792 svn://svn.berlios.de/wsjt/branches/wspr wspr
C:\tmp> cd wspr
C:\tmp\wspr> copy Makefile.MinGW Makefile

Depending on the versions and installation paths of Python and
PyInstaller, you may need to edit "Makefile".  For example, I changed
"python25" to "python27" in several places, and "pyinstaller-1.4" to
"pyinstaller-1.5.1".  In addition, you should remove the "\" at the end
of this line:

all:    WSPR.EXE fmt.exe fmtave.exe fcal.exe fmeasure.exe wspr0.exe

and delete the following line.  Then, just type "make" to build wspr.exe

c:\tmp\wspr> make
