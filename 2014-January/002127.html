Hi Greg,

I've not seen that site be for, looks pretty interesting.

Building the docs is a quick 1 minute task. I build a number of projects
every night with cron jobs.

With a few script lines, anyone could build the html. Could also add
commit hooks directly too the BerliOS repo.

However, transferring the docs to physics.princeton.edu/pulsar/K1JT
requires web-server access, which most of us don't have.

A simple solution, create a workstation daily task (cron);

Add cron job, use whatever time ya like (say, nightly @ 0115 local);
15 1 * * * /path/to/build/sync-script.sh

In the script:
* svn export .. .. /wsjtx/doc   # only need wsjtx/doc/
* cd ./wsjtx/doc && ./build-doc.sh toc2
* rm -rf source/

Then, I'd use rsync between workstation and the web-server:

* rsync -azvr wsjtx/doc/ /path/to/web-server/wjtx/doc/

You could use scp, sftp or any number of alternatives, but rsync keeps
the file count down, and only transfers incremental diffs. The whole
process would not take more than a couple minutes tops.
