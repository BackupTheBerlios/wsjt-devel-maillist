Hi John,

Thanks for catching the need for declaring db() as a single-precision 
function in flat3.f90.

As for the apparently duplicated array name in calls to flat3(): this is 
intentional.  It allows for the case in which the resulting "flattened" 
array is passed back to the calling routine in a different array from 
the original one.  In the present use, when the two arrays are the same, 
the original data are overwritten.
