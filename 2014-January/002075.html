Hello All,

I've updated build-doc.sh. Tested on Ubuntu 12.04.3 / 14.04.

4 - Build Modes: No TOC, Top TOC, Left TOC and ALL, but it's now user
controlled.

To build wsjtx-main.html, without toc:
./build-doc.sh

To build wsjtx-main-toc1.html, top toc:
./build-doc.sh toc1

To build wsjtx-main-toc2.html, left side toc:
./build-doc.sh toc2

To build all wsjtx-main*.html, all three methods:
./build-doc.sh all

If anything other than those 4 options are used, you should get a little
error message listing the build methods available.

There are many things that need to be done in the long run, but that
will take some coordination with the commiters, single main file v.s.
chapter builds, rig files, etc etc. For now, this should give users 4
easy options.

If you have any issues running it, shoot me a note, it's a simple
script, should be able to sort it out quickly. Be careful of line wraps.
