The .INI file is personal to your system. WSJT-X will create a vanilla 
.INI file itself if there is none present. Default values are provided 
for all entries where it makes sense to do so.

The value default process is at the individual entry level; so it is 
possible to return entries to their default value by simply editing the 
.INI file and removing the entry in question.

In an update to come soon after the current beta version goes to a full 
release; the serial port fields in the configuration dialog will become 
directly editable, so there will be no need to edit the .INI file 
directly, to use a non-standard communications port.
