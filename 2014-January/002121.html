>> There has been an obscure bug reported for Qt v5 that causes 
>> program freezes on some setups when a screen saver or login 
>> screen kicks in. One known symptom is that it only affects the 
>> program that has focus. If you regularly leave WSJT-X running 
>> unattended; you could try minimizing it (main window & waterfall) 
>> and see if the freeze still occurs. It may also be possible to reproduce
>> it on demand by typing CTRL+ALT+DEL to bring up the login/task-
>> manager screen.

Interesting.

In this case I do have a screen saver setup -  but forgot all about that ...
probably because the power settings turn off the monitors before the screen
saver was being invoked.

I tried to repro a couple of times after receipt of your note - but not able
to.  These actions had no adverse effect on the program:

- CTRL-ATL-DEL and just let that screen sit there
- CTRL-ALT-DEL and lock the computer
- let the computer invoke the screen saver (10 minutes)
- force the screen saver to run ("preview")

I did the aforementioned test with both the main WSJT-X window having focus
... as well as the Wide Graph window having focus.
