Ok, now I understand where and what the issue is. After working
through several options on my desktop, I installed Cygwin up front on
my WinXP laptop, and built the html inside.

After install, Cygwin has a small footprint ~260 MB or so, and does
not interfere with any HOST OS development package, e.g. Perl, Python
Qt etc, which was the primary reason for using it over a native
package install method.

When selecting source-highlight in the Cygwin pkg manager, I did not
notice an excessive number of packages being installed, then again,
this was an installation, so I wasn't watching that closely.

> 
> If we move compiling instructions to a separate manual, which I
> think is desirable, I can live with it.  Or I can build source
> highlighter from source, on Windows; but that requires installing
> "boost".  I can do that, too...

Or another solution is to "not" use source code highlighting, but I
find is useful as a reader.

The whole idea was to make it easy for folks to get started with
document contributions (another reason why Cygwin was an easy choice).
Installing from source doesn't seem the best way, but certainly a
workable solution. I spend the majority of my time in a Linux
environment, admittedly I'm not up to speed on all the porting issues.
