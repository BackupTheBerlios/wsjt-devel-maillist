There has been an obscure bug reported for Qt v5 that causes program 
freezes on some setups when a screen saver or login screen kicks in. One 
known symptom is that it only affects the program that has focus. If you 
regularly leave WSJT-X running unattended; you could try minimizing it 
(main window & waterfall) and see if the freeze still occurs. It may 
also be possible to reproduce it on demand by typing CTRL+ALT+DEL to 
bring up the login/task-manager screen.
