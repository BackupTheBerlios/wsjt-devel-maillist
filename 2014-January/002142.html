Further info after trying on my Ubuntu system.

It was the 32-bit version of libgfortran.so.3 that was missing. Fixed by:

sudo apt-get install libgfortran3:i386

> This is not an optimum solution but should give you a work around.
>
> 64-bit Intel Linux is perfectly happy running 32-bit binaries, in fact 
> IMHO there is no convincing reason to have 64-bit binaries for small 
> programs that don't need 64-bit registers or pointers. Nevertheless 
> Linux distros often don't have the 32-bit system libraries that 32-bit 
> binaries link dynamically to. You can usually install the necessary 
> 32-libraries to get things working, something like:
>
> sudo apt-get install ia32-libs
>
> or your distros equivalent. You can always remove the libraries when a 
> 64-bit build of kvasd is available.
