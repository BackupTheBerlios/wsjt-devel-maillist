No.  To have commit privileges to the SVN repository, you need to 
register at Berlios.

Go to http://developer.berlios.de/projects/wsjt/ ,
click on "New User via SSL" at the top left, and get yourself a login 
name (maybe your callsign?) and password.  Then I can add you to the list.

We will need to decide how to manage the documentation within the 
repository.  Greg (KI7MT) outlined some possibilities yesterday: for 
example,

"should the documentation reside within the project branch
(e.g. ../wsjt/trunk/wsjtx/docs), or be at the trunk level
(e.g. /wsjt/trunk/wsjt-docs/{wsjt,wsjtx,wspx,wsprx,map65})
etc.?"

I don't have strong opinions about this, but up to now the documentation 
for each project branch (WSJT, MAP65, WSPR, WSJT-X, WSPR-X) has been 
separate and distinct.  Yesterday I committed
the KI7MT documentation directory and its contents to the wsjtx branch 
of the repository.  It includes a significant amount of first-pass 
editing of the main source file, wsjtx-main.txt -- so others should 
start there, NOT with Greg's original file.

We will need to work out a way to keep track of who is working on 
particular sections, so we don't step on each other's toes.
