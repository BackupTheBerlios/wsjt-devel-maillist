To build:
cd to where you downloaded doc.tar.gz
tar -xzvf ./doc.tar.gz
cd ./doc && chmod +x ./build-doc.sh && ./build-doc.sh all

I pulled the latest SVN 3655, and re-worked wsjtx-main.txt to split
each sub-section into it's own file. This will allow a build-check
script to parse the files (./source/*.txt) and provide the user with
the current status. I've edited very little content, apart from minor
syntax items here and there. I did add a sub-section for Installation
>> Mac OS. For the best view of changes, toc2 seems the best.

The HTML files render properly for all three TOC methods, post split.
The tar file contains all the files apart from build-check script,
which is not finished yet. I also hard coded the doc version for
testing outside the wsjtx source tree.

The idea is, one can pull the latest SVN, run a quick status check and
be provided with a list of files / topics which need work (added,
updating, deleting, etc) based on the file's tag. I should have status
check script complete tomorrow evening. Current status of the changes
I am proposing:

COMPLETED ITEMS:
* Cleaned up a few typo's, but mostly focused on structuring the files.
- -- To accommodate a distributed work-flow. (54+ files added to
./source/...txt).

* Turned wsjtx-main.txt into a control file, using
include::<file-name> directive to pull in sub-sections. A few small
sections remain, but they too could be be pulled into working files.
- -- Allows folks to work on the smaller sub-sections, rather than
everyone trying to work the same doc (wsjtx-main.txt).
- -- It also helps keep HTML [[X??]] anchors from having to be
re-written when a section changes, gets added or removed.

* Added '// Status=.. ..' Tag to the top of each file, to enable
build-check.sh script functions.
- --  Tested several tags + file combinations using, CURRENT, REVIEW,
EDIT, INPROGRESS, OBSOLETE,.
- -- Tags can be changed, added to, reduced or whatever is needed.

TO-DO ITEMS
