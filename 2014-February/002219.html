Greg --

I think I've become a believer that we should make the .../doc directory 
a separate branch from the code-development branches.

This probably means that where we now have

   ...branches/wsjtx/doc/...

we should instead have

  .../branches/doc/wsjtx/...

and also, in due course, things like

   .../branches/doc/map65/...

etc.

My notion was that we have improved the User's Guide significantly since 
the v1.3, r3673 release.  These improvements apply specifically to v1.3, 
r3673, and users should have benefit of the better, more complete 
information.  At least for now, it makes sense to me to post these 
improvements as they are made.  I think nearly everyone that uses our 
manual fetches it when needed, online.

	-- Joe
