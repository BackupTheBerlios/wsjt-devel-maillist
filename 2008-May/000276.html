Please no. Lets not. In fact, I want to remove it from wsjt.

in wspr
./configure --help
  --with-portaudio-include-dir=<path>
                          path to portaudio include files
  --with-portaudio-lib-dir=<path>
                          path to portaudio lib files

Remember there is _still_ a portaudio v18 being used by other programs,
if you compile up v19 and put it into /usr instead of v18, you
break the other programs. There was some talk from the portaudio
folks this winter about finally making a final release, which would
mean folks would start migrating their code to v19; but it seems
that effort has stalled.

On FreeBSD
./configure --with-portaudio-include-dir=/usr/local/include/portaudio2 \
--with-portaudio-lib-dir=/usr/local/lib/portaudio2

On your system, fill in the right stuff, then try the compile.

Yes, which is what the new configure/Makefile does.

(e.g. On FreeBSD it is in /usr/local/lib/python2.5/site-packages )

So you can run wsjt outside of the build directory.
