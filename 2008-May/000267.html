The FreeBSD wsjt package uses the portaudio 19 package as well rather
than the copy we include. This is precisely what I am trying to state.
Once we have a known stable release of v19 that all the distributions agree
to use, we can remove the included portaudio v19. In fact I'd like
to remove it now. The only difficulty is knowing where distributions
are going to put portaudio v19. It is also very possible that all the
packages that now depend on v18 will recode to use v19 and it will
not break things to default to /usr/include/portaudio or
/usr/local/include/portaudio, expecting v19 but getting v18 include files.
