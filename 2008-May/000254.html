This definitely applies to Mandriva 2008.0 & 2008.1 x86_64
It appears the portaudio section is very fussy which fortran compiler
it uses.
Use of g95 fortran compiler will result in audio.so not being built.
For Mandriva 2008.xx86_64 use the gfortran compiler.
For some reason I don't understand gcc-gfortran is found
when ./configure is run gcc4.3-gfortran is ignored.
This probably will apply to most linux distros
