Joe Taylor wrote:
> I've had several reports recently of people trying to use a 
> COM port numbered COM10 and above to do the PTT task for 
> WSJT.  This doesn't work, and it turns out to be a little 
> known (at least unknown to me) peculiarity deep inside the 
> Win32 API.
>
> Anyway, there is supposedly a work-around, and I have now 
> compiled it into WSJT7 r1017.  Except to see that it still 
> works COM port numbers 9 and below, I haven't tested it -- 
> since I don't have a machine configured with a COM port 
> above 10.
>
> If you *do* have such a machine, please download and test
>
> http://physics.princeton.edu/pulsar/K1JT/WSJT7r1017.EXE
>
> Place this file in your WSJT installation directory and 
> start it instead of your existing WSJT7.EXE.  If it works OK 
> with the large COM port numbers, please let me know!  ... 
> and you can then rename the file as WSJT7.EXE, replacing 
> your existing file of that name.
