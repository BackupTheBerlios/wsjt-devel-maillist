The -fPIC is to generate position independent code, not platform
independent. On the x86 series of chips it permanently reserves a
register which causes a slow down due to the lack of registers to start
with, the poor optimiser doesn't stand a chance! It is ONLY used for
creating shared objects and serves no purpose for static linking, so
get rid of it. -fPIC is used with -DPIC. To make the code platform
dependent use one of the -mXXX compiler directives on GCC.
