On Tue, Jan 02, 2007 at 07:12:12PM -0500, Joe Taylor wrote:
> Would it also be best, in your opinion, to change the default audio 
> device to "1" for both input and output?

I don't think there is a sensible default - it depends on which audio
layer the user has compiled in (ALSA, OSS or portaudio (and which
options portaudio was compiled with)) etc. On my system I am using
device 11.

If possible it would be useful to present the list of devices within the
GUI (not just in the startup messages), and perhaps make the user select
a device when they run wsjt for the first time.

With regard to startup messages, usually they are not seen when programs
are launched through the desktop's menu system (GNOME/KDE etc). For
Debian I have the menu entry for wsjt specifically launch an xterm so
that the startup messages can be seen.
