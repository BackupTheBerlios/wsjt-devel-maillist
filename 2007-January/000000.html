Dear Colleagues,

Recently I had a request from a friend for an 
easy-to-install WSJT package for Linux.  I made him a 
tarfile named wsjt6_r315.tgz that contains the following files:

CALL3.TXT
KVASD
libBLT.2.4.so.8.4
libg2c.so.0
libsamplerate.so.0
libtcl8.4.so.0
libtk8.4.so.0
RxWav/Samples/W8WN_010809_110400.WAV
TSKY.DAT
UpdateHistory.txt
wsjt
wsjt6
wsjtrc
WSJT_User_600.pdf

If you download this file from 
http://pulsar.princeton.edu/~joe/K1JT/wsjt6_r315.tgz into 
your login directory, you can install and run WSJT 5.9.6 
r315 with the following three commands:

   % tar xzf wsjt6_r315.tgz
   % cd wsjt6_r315
   % ./wsjt

I have tested this procedure on a fairly old Red Hat 9 
installation, on pristine copies of Knoppix 5.0 and Ubuntu 
6.10, and on a recent Debian install.  It seems to work fine 
on all of these distributions.  It looks best on the Knoppix 
and Debian-based systems, but that's probably because I was 
using those when I selected the default fonts, etc.

I believe that everything necessary for running WSJT will be 
installed into the directory ~/wsjt6_r315.  To uninstall it 
you can just type

   % rm -rf ~/wsjt6_r315

I would appreciate it if others on the WSJT Developer's List 
would give this installation procedure a try, and let me 
know how it works (or fails to work).  (Diane: It might even 
work on FreeBSD ... that would be a real bonus!)

If you install it somewhere other than your default login 
directory, you will need to edit the first line in the 
startup script "wsjt".

	-- 73, Joe, K1JT
