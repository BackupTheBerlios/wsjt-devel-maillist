This is our well-known problem with gfortran.  (See the wsjt-devel 
traffic earlier today, and earlier as well.)  Probably the same kind of 
fixes as those made today to WSJT will correct it.

For now, if you want a working program, use g95.  Or use the 
"production" version, on the WSJT web site, which is compiled with 
Compaq Visual Fortran.

MAP65 has a few rough edges.  The number of users is probably a few 
dozen, as opposed to the several thousand users of WSJT and WSPR.  As a 
consequence, given my limited time I don't feel the need for nearly as 
much "polish" as is warranted on the other programs.

I've been a heavy user of MAP65 for nearly two years, and a very happy 
one.  The program provides amazing capabilities for EME.
