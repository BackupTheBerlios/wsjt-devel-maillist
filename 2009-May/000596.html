as far as I'm concerned Ubuntu 9.04 was a disaster.
lots of thing don't work, some very serious. 
For example executing a binary file in the same directory its in :-
you would type at CLI  ./<binary name>
ubuntu 9.04_x64 replies with no such file or directory
the 32 bit version may be better, I don't know.
However, on the startup crash it may be worth checking which device
number wsjt it trying to startup with. I found with ubuntu that it
would crash immediately if the device number was not the default.
I guess you can edit wsjtrc to set the audio device to the default.
I also found that clicking on the "logqso" or the "add" button caused a
crash. it weas only last month that a g95 deb was released for AMD64
machines, up until then wsjt just would not compile on a 64bit machine
running the Ubuntu_AMD64 variant.
