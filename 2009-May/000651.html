You wrote:
--
I can compile and make the latest wspr on osx 10.5.6 intel but at run-
time I get the following:

WSPR Version 1.12_r1200 , by K1JT
Run date:   Sat May 16 16:43:51 2009 UTC
alloc: invalid block: 0x660338: b0 0 0

I suspect the problem is with portaudio.   What you should get is:

WSPR Version 1.13_r1246 , by K1JT
Run date:   Tue May 19 09:54:12 2009 UTC

Audio     Input    Output     Device Name
Device  Channels  Channels
------------------------------------------------------------------
    0        2         0       Built-in Microphone
    1        2         0       Built-in Input
    2        0         2       Built-in Output

User requested devices:  Input = 0   Output = 0
Default devices:         Input = 0   Output = 2
Will open devices:       Input = 0   Output = 2
******************************************************************

What you have to do is to start putting print statements in the  
code.  I suggest you start with wspr2.F90  just before it calls   
padevsub.   This routine begins the portaudio initialization where  
the " Audio     Input    Output     Device Name" titles are shown.     
You have to find out just where the code is crashing.
