Hi Kari and all,

You are correct that I neglected to mention two packages 
that must be installed to compile and run wsjt on *ubuntu.

For the record, the full list of packages required after a 
clean install of Xubuntu (and presumably also Ubuntu, 
Kubuntu, ...) is:

	subversion
	python2.6-dev
	python-pmw
	python-numpy
	python-imaging-tk
	portaudio19-dev
	libfftw3-dev
	libsamplerate0-dev

Then, of course, you must install either gfortran or g95.

I had a bright idea this morning that maybe the 
gfortran-compiled code fails only on multi-core CPUs... so I 
tried again on an IBM T40 laptop.  No joy, with gfortran it 
still crashes on startup; with g95 all is well.

As far as I can tell, compiling WSJT or WSPR with gfortran 
works only in Finland.  At least, it does not work in New 
Jersey.  :-)

Therefore, the recommended procedure is to compile using 
g95.  Then it's a simple procedure -- just

$ ./configure --enable-g95
$ make
$ python wsjt.py  (or python wspr.py)

and the programs run without problems.

	-- 73, Joe, K1JT
