On 04/06/13 08:01, John Nogatch wrote:
> With much help from Kamal KA6MAL, wsjtx packages for Ubuntu 12.04,
> 12.10, & 13.04 are available:
> https://launchpad.net/~jnogatch/+archive/wsjtx
>
> sudo add-apt-repository ppa:jnogatch/wsjtx
> sudo apt-get update
> sudo apt-get install wsjtx
>
> Debian packages are in the queue, but not available yet.
>
> It would be nice if one of you developers could verify installation,
> before it is announced to the broader audience.
>
> Note that package files are installed under /usr/share and /usr/lib,
> and the other files which were in wsjtx_install are in ~/.wsjtx/
>
> 2 modifications were made, which have not yet been checked into the svn source:
>
> 1. /dev/ttyS4-7 were added as choices in the Configuration menu.
>
> 2. When no sound in/out devices have been saved in wsjtx.ini,
> Pa_GetDefaultInputDevice() and Pa_GetDefaultOutputDevice() are used to
> determine the devices. This allows wsjtx to come up the first time,
> without displaying error pop-up dialogs. This is an imperfect
> solution, because the corresponding index into the menu lists (which
> do not yet exist!) is not determined. It would be better if the name
> of the sound device was saved in wsjtx.ini because list numbering is
> not stable if USB soundcards are plugged/unplugged.
>
> I find 13.04 has severe graphics problems on my old laptops, with i915
> Intel graphics. I found 12.04 and 12.10 to be better, on the older
> laptops.
