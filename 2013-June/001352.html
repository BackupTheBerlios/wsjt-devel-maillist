Hi John,

AC6SL wrote:

> I have made enough JT9 QSOs that I now want to avoid working the same
> callsign again, on a per band basis, i.e. "dupe checking".

Thanks for sharing your ideas on such a feature.  MAP65, sister program 
to WSJT-X, already has this feature.  Its motivation was my need for 
dupe checking in EME contests.  The program maintained a Python 
"dictionary" of calls already in the log.  Unworked calls appear in the 
Band Map flagged with an asterisk.  Some vestigial remains of this MAP65 
code are in the WSJT-X code around line 270, but the dupe checking was 
removed.  It could easily be replaced and enhanced to include 
band-by-band checking, if there is demand.

I think most WSJT-X users on Windows are using JT-Alert, which does this 
dupe checking and much more.  So the demand for a built-in feature of 
this sort is not strong, from Windows users.

     -- Joe, K1JT
