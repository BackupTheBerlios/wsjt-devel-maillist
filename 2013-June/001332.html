> Hi!
>
> Maybe it is of interest for someone.
> When starting WSPRX from a terminal window it finds the correct path  
> for storing the hashtable.txt and ALL_WSPR.TXT.
>
> When starting via unity starter panel it is necessary to include the  
> "Path" statement in the configuration file. Otherwise these files  
> are stored directly in the users home directory. The resulting error  
> is not uploading spots to the database.
>
> Example of wspr.desktop file:
>
> ---- START ----
> #!/usr/bin/env xdg-open
>
> [Desktop Entry]
> Version=1.0
> Type=Application
> Terminal=false
> Icon[de_AT]=/home/michael/wsprx/wsjt.ico
> Name[de_AT]=WSPR-X
> Exec=/home/michael/wsprx/wsprx
> Path=/home/michael/wsprx
> Name=WSPR-X
> Icon=/home/michael/wsprx/wsjt.ico
> ---- END ----
>
